# Padrões de Vendas Reais - Análise de Conversas Centauro

> Extraído de `wpp-analytics.json` - Conversas reais bem-sucedidas

## 📊 Padrões Identificados de Atendimento de Sucesso

### 1. **Tom de Comunicação Direto e Prático**

**Características observadas:**

- Respostas curtas e objetivas
- Sem "enrolação" - vai direto ao ponto
- Linguagem coloquial mas profissional ("mano", "blz", "tá certo")
- Evita explicação excessiva

**Exemplos reais:**

```
Cliente: "Tem tênis número 27 para menino?"
Atendente: "Temos sim. Gostaria que eu deixasse reservada?"

Cliente: "Tem chuteira n31 campo?"
Atendente: "Temos sim, 3 cores. Qual seria a numeração?"
```

**❌ Evitar:**

- "Olá! Seja muito bem-vindo(a) à Centauro! Como posso ajudá-lo hoje? Temos ótimas opções..."
- Respostas longas com informações não solicitadas

---

### 2. **Fechamento Rápido - Identificar Intenção**

**Padrão observado:** Em conversas de venda, o atendente rapidamente identifica:

1. Produto desejado
2. Numeração/tamanho
3. Disponibilidade
4. Próximo passo (reservar ou vir na loja)

**Exemplo de fluxo eficiente:**

```
[1] Cliente: "Boa tarde, Vocês tem qual pro 5 bump disponível na loja?"
[2] Atendente: "Boa tarde. Qual seria a numeração?"
[3] Cliente: "42 se a forma for grande"
[4] Atendente: "Temos sim, 3 cores. Gostaria que eu deixasse reservada?"
[5] Cliente: "Blz"
[6] Atendente: "Me mande um oi quando estiver vindo. Estou te esperando"
```

**Tempo médio para fechar intenção:** 3-5 mensagens

---

### 3. **Confirmação de Disponibilidade Antes de Prometer**

**Padrão correto:**

```
Cliente: "Tem chuteira n31 campo?"
Atendente: "Só um momento, vou verificar no sistema"
[...consulta...]
Atendente: "Temos nas cores X, Y e Z"
```

**Nunca dizer "temos" sem confirmar!**

---

### 4. **Direcionamento para Ação Concreta**

**Ações finais típicas:**

- "Pode vir buscar, estarei te esperando"
- "Deixe seu telefone que te aviso quando chegar"
- "Me chama quando estiver vindo que deixo separado"
- "Para encomendar, precisa vir em loja ou pedir no site"

**Evitar deixar o cliente sem saber o que fazer:**

```
❌ "Ok, temos o produto."
✅ "Ok, temos o produto. Quer que eu separe para você buscar?"
```

---

### 5. **Uso de Imagens para Produtos**

**Quando o cliente pede:**

```
Cliente: "Manda foto"
Atendente: [envia foto do produto]
Atendente: "Esse é o modelo X. Gostou?"
```

**Padrão observado:** Atendentes enviam fotos rapidamente quando solicitado, sem textão.

---

### 6. **Transparência sobre Limitações**

**Exemplos honestos:**

```
Cliente: "Vcs têm camisa do Flamengo Feminina?"
Atendente: "Boa tarde, somente para pedir direto no site"

Cliente: "Consigo comprar assim por aqui pelo WhatsApp?"
Atendente: "Não fazemos venda via link, somente em loja ou direto no site"

Cliente: "Tênis número 27 para menino?"
Atendente: "So temos na numeração 35"
```

**Nunca prometer o que não pode entregar!**

## 🎯 Padrões de RESOLUÇÃO DE PROBLEMAS (SAC)

### 1. **Pedir Informações Específicas**

**Sempre coletar:**

- Número do pedido
- CPF do cliente
- Nome completo
- Descrição clara do problema

**Exemplo:**

```
Cliente: "Não recebi meu pedido"
Atendente: "Qual o número do pedido e CPF?"
```

### 2. **Abrir Chamado com Dados Completos**

**Formato observado:**

```
Ticket #XXXXXX
Pedido: XXXXXXX
CPF: XXX.XXX.XXX-XX
Problema: [descrição clara]
Ação necessária: [específica]
```

### 3. **Prazos Realistas**

```
Cliente: "Quando resolve?"
Atendente: "Aguarde o retorno do time responsável. Prazo de até 48h"
```

---

## ⚠️ O que NÃO fazer (erros identificados)

### 1. **Respostas robóticas/genéricas**

```
❌ "Olá, me chamo Bolt, sou agente virtual do Grupo SBF! Para que eu possa..."
✅ "Oi! Qual sua dúvida?"
```

### 2. **Explicações excessivas**

```
❌ "Nosso sistema funciona da seguinte maneira: primeiro você precisa acessar o portal, depois clicar em X, então aparecerá Y..."
✅ "Precisa do seu login. A senha é as 4 primeiros dígitos do CPF"
```

### 3. **Deixar cliente em aberto**

```
❌ "Vou verificar e te retorno"
[sumiu por 2 horas]
✅ "Vou verificar agora. Me dá 2 minutos"
```

## 📋 Checklist de Atendimento Eficiente

### Para VENDAS

- [ ] Identificar produto desejado
- [ ] Confirmar numeração/tamanho
- [ ] Verificar disponibilidade
- [ ] Propor ação (reservar/vir buscar/pedir no site)
- [ ] Confirmar próximo passo

### Para SUPORTE

- [ ] Coletar número do pedido
- [ ] Coletar CPF do cliente
- [ ] Entender o problema
- [ ] Abrir chamado se necessário
- [ ] Dar prazo realista de retorno

## 🔥 Insights Principais

1. **Menos é mais** - Respostas curtas funcionam melhor
2. **Ação imediata** - Sempre propor próximo passo
3. **Transparência total** - Se não tem, diga que não tem
4. **Dados precisos** - Sempre coletar pedido e CPF para suporte
5. **Humanização leve** - Tom casual mas profissional
